This is a project that I had started while on deployment. It took me quite a while to get to this point because I did not have a lot of free time to work on the project while I was overseas. However, I am happy with the outcome and was extremely excited with what I acheived.

How this is supposed to be used:

Click Add Belt to bring up the form. Insert the belt rank by color (i.e. White). On the second line input how many stripes that belt has (i.e. 5). Finally, on the last line input how many classes it takes to get a stripe (i.e. 10). Submit the form, and then start clicking up. One click up represents one class attended.

So if a user input that a White belt has 5 stripes, and that it takes 7 classes to earn one stripe, then after 35 up clicks (or classes attended), all stripes would be earned and the belt can be ranked up. Or, after 7 clicks up the belt would earn 1 of 5 stripes, after 7 more clicks up (14 classes) the belt would earn the 2nd stipe of 5, and so on until they have gotten all 5 stipes.

If you do manage to look at my code, I must warn you beforehand because I was all over the place with trying new functions and creating variables. I do plan to refactor the code to make it more readable and less redundant, but that will take some time.

Additionally, I do plan on making this into an actual CRUD App using Java, Spring, JS, CSS, HTML, and more, but for the meantime I hope this demonstrates my capabilities with Document Object Model (DOM) manipulation.
